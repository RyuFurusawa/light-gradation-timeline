# 光のグラデーション・タイムライン (Light Gradation Timeline)

色彩演習のための、時間変化する光をブラウザ上でデザインするツールです。円形のタイムライン上に色点を配置し、色と色のあいだをどう補間するかを8つの方式から選んで観察・再生できます。

**HDR対応・単一HTMLファイル**。依存はCDN経由のp5.jsのみで、ビルド不要です。開いた環境に応じて、真のHDR表示 → Display P3広色域 → sRGBの3段階に自動でフォールバックします。

## 使い方

このリポジトリを [GitHub Pages](https://pages.github.com/) で公開している場合は、そのURLをブラウザで開くだけで動作します。ローカルで使う場合は `index.html` をブラウザにドラッグ＆ドロップしてください（インターネット接続はp5.jsの読み込みに必要です）。

- 色点をドラッグ＝タイミングを変更
- 色点を円の外へドラッグ＝削除（最低2点は残ります）
- 色点を2回タップ／ダブルクリック＝カラーピッカーで色を変更
- リングの帯をタップ＝色点を追加
- 白い針（再生ヘッド）をドラッグ＝再生位置を変更
- 左上のハンバーガーメニュー＝補間方式の選択
- 右上の「i」＝アプリ内の使い方・注意事項パネル

## 補間方式（8種）

数学的な選択というより、**色をどう「モデル化」するか**という立場の違いです。

1. **ステップ切替** — 補間なし。瞬時に次の色へ切り替わる
2. **HSB色相回転** — 彩度は保たれるが、色相環は等明度ではないため明滅する
3. **RGB中間混色（光量平均）** — ガンマを解いた物理的な光量の平均
4. **RGB中間混色（画素値平均）** — sRGB符号値の単純平均（光量の平均ではない）
5. **OKLab 色相回転** — 知覚均等性を目指した色空間（[Björn Ottosson, 2020](https://bottosson.github.io/posts/oklab/)）での色相回転
6. **OKLab 中間混色** — OKLab空間上の直線補間
7. **CIELab 色相回転** — CIELab（1976）での色相回転。青の領域で色相が非線形に曲がる既知の癖がある
8. **CIELab 中間混色** — CIELab空間上の直線補間

## 検証機能

- **輝度検証** — 選択中の方式が主張する「明度」の平均で、無彩色の基準グレーを表示
- **xy色度図** — CIE 1931 xy色度図に色点・現在色・補間の軌跡を表示。xy図のみ表示時は、図上の点をドラッグして色を直接編集可能（明るさは保持）
- **知覚明度(OKLab)で統一** — 全色点のOKLab明度を最低／平均に揃え、Helmholtz–Kohlrausch効果を観察できる
- **彩度最大** — 色相・明度を保ったまま、色域の許す限り彩度を最大化（無彩色はそのまま）

## 対応環境

- **HDR表示**：WebGPU＋`rgba16float`＋拡張トーンマッピングに対応したブラウザ・OS・ディスプレイ（例：最新のSafari／Chrome＋HDR対応ディスプレイ）
- **Display P3**：上記が使えない場合、広色域ディスプレイであればP3で表示
- **sRGB**：どちらも使えない環境では通常のsRGB表示にフォールバック

現在の表示モードは画面右下の診断表示（`HDR` / `P3広色域` / `sRGB`）で確認できます。

## 授業での利用

色彩演習の教材として、以下のような検証課題に使えます。

- 補間方式ごとの色度図上の軌跡の違い（直線・弧・跳躍）
- 無彩色に色相はないこと（powerless hue）
- Helmholtz–Kohlrausch効果と色知覚の個人差
- 照明による物体色の変調と色の恒常性（黒布とクレヨンを使った制作課題と組み合わせ可能）

詳しい実験課題例は [`docs/lesson-plan.md`](docs/lesson-plan.md) を参照してください（WorkFlowy互換のMarkdown形式）。

## 技術メモ

- 色変換（sRGB ⇄ OKLab ⇄ CIELab ⇄ CIE xyY）はすべて本ファイル内に実装
- 無彩色（彩度がεを下回る色）の色相は「無力（powerless）」として扱い、[CSS Color 4](https://www.w3.org/TR/css-color-4/) の規定に沿って補間時の色相の迷走を防止
- HDR描画は生のWebGPU、UI・リング・xy色度図はp5.jsという役割分担のハイブリッド構成
